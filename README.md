# mkup-guide

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面资料的整理与公开访问入口。

## 项目简介

mkup-guide 旨在收集、整理并长期维护一组独立的 HTML 页面。这些页面可能包含各类说明、工具、参考或示例内容，以纯前端形式呈现，无需后端支持。

仓库采用简单的目录结构管理页面，确保每个页面均可独立访问，互不干扰。适合需要长期保存或分享的静态文档场景。

## 目录说明

```
mkup-guide/
├── README.md          # 本文件
├── index.html         # 仓库首页 / 导航入口
├── pages/             # 存放所有独立 HTML 页面
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
└── assets/            # 共享资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名应简洁且有描述性。
- `assets/`：存放页面间共享的静态资源，避免重复引入。

## 页面归档说明

- 所有页面均为纯 HTML + CSS + JavaScript，不依赖外部 API 或数据库。
- 页面之间无强关联，可单独浏览。
- 鼓励在页面内使用相对路径引用资源，以保证仓库克隆后仍可正常使用。
- 如需添加新页面，请在 `pages/` 下创建文件，并在 `index.html` 中添加链接。

## 维护说明

- 本仓库由维护者定期更新，内容以归档和整理为主。
- 欢迎提交 Issue 或 Pull Request 添加新的独立页面，或改进现有页面。
- 提交前请确保页面内容完整、无外部依赖链接失效、无恶意代码。
- 不建议在页面中包含任何形式的推广、营销或诱导点击内容。
